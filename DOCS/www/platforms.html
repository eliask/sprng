Platforms to which SPRNG has been Ported

We describe, in this section, certain platform dependent matters relating to SPRNG installation and usage.

We also give typical compilation examples. These examples assume that the SPRNG library liblcg.a is in the parent directory of the user's source code, and that the the SPRNG header files are in the subdirectory SRC of the parent directory.

We give examples with MPI in the case of parallel machines and sequential examples in the workstations. In order to execute the MPI executables, the user typically issues a command of the following form:

mpirun -np n a.out

where n is the number of processors required.

Parallel Machines

Cray T3D

SPRNG has been ported to the CRAY T3D. The particular system tested on was the one at the Pittsburgh Supercomputer Center.

Typical Compilation:

  • /mpp/bin/cc -Tcray-t3d -O 3 -I../SRC -I/usr/local/mpp/include program.c -L.. -llcg -L/usr/local/mpp/lib -lmpi

  • (1) cc -E -P -I../SRC -I/usr/local/mpp/include program.F
    (2) mv program.i program.f
    (3) cf77 -O scalar3 program.f -L.. -llcg -L/usr/local/mpp/lib -lmpi
A three step procedure is used in compiling FORTRAN programs. The first step uses the C preprocessor to produce a file with a .i extension. We move this to a file with a .f extension and compile as a standard FORTRAN program.

The system on which we tested SPRNG does not use mpirun. Instead, we set the number of processors to be used by issuing a command of the following form:

setenv MPP_NPES n

where n is the number of processors. The executable can then be run by just entering its name.

IBM SP2

SPRNG has been ported to the IBM SP2 available at the Cornell Theory Center. The xlc compiler defines a macro called _LONG_LONG which indicates that the type long long is available. The use of this type while installing SPRNG enables a faster implementation of the Linear Congruential Generators.

Typical Compilation:

  • mpcc -O3 -qstrict -qarch=pwr2 -qlanglvl=ansi -qtune=pwr2 -I../SRC -I/usr/lpp/ppe.poe/include program.c -L.. -llcg

  • mpxlf -O3 -qstrict -qarch=pwr2 -qtune=pwr2 -WF,-I../SRC -WF,-I/usr/lpp/ppe.poe/include program.F -L.. -llcg
The C and FORTRAN compilers used above automatically link to the MPI library. The -WF, switch is used to pass flags in a format similar to that with "standard" compilers.

HP/CONVEX Exemplar

SPRNG was ported to the HP/CONVEX Exemplar at NCSA.

Typical Compilation:

  • cc -O2 -I../SRC program.c -L.. -llcg -L/usr/convex/mpich/lib/hpux/ch_hpmp -lmpi

  • (1) cp program.F program.f
    (2) fc -fpp -O2 -I../SRC program.f -L.. -llcg -L/usr/convex/mpich/lib/hpux/ch_hpmp -lmpi
We have assumed that the user's FORTRAN program is in a file with the extension .F. If, however, the user's file already has the extension .f, then the first step above need not be performed.

SGI-CRAY Origin 2000

SPRNG has been ported to the SGI-CRAY Origin 2000 at NCSA. Please see the notes for the SGI Power Challenge for further details.

Typical Compilation:

  • cc -O3 -align64 -64 -I../SRC program.c -L.. -llcg -L/usr/lib64 -lmpi

  • f77 -O3 -align64 -64 -DPOINTER_SIZE=8 -I../SRC program.F -L.. -llcg -L/usr/lib64 -lmpi

SGI Power Challenge

SPRNG has been ported to the SGI Power Challenge at NCSA. Both 32 bit and 64 bit compilations can be performed. The 64 bit version is faster since it enables an efficient implementation of the Linear Congruential Generators. The file make.SGI contains variable definitions for both the versions. Initially, we comment out the variable setting for the 32 bit version. If users wish to compile SPRNG in the 32 bit mode, then they should uncomment the lines that refer to the 32 bit version and comment out the lines that refer to the 64 bit version.

Typical Compilation:

  • cc -O3 -align64 -64 -I../SRC program.c -L.. -llcg -L/usr/lib64 -lmpi

  • f77 -O3 -align64 -64 -DPOINTER_SIZE=8 -I../SRC program.F -L.. -llcg -L/usr/lib64 -lmpi

Sequential Machines

SPRNG has been ported to the workstations mentioned below. Since these machines were single processor systems, the MPI version of SPRNG was not used.

HP

SPRNG was been ported to an HP9000/715 workstation.

Typical Compilation:

  • c89 -O -I../SRC program.c -L.. -llcg

  • fort77 -O -I../SRC program.F -L.. -llcg -L/usr/lib -lU77
We need to use -llcg -L/usr/lib -lU77 only if command line arguments are used in the program.

PC running Linux

SPRNG was been ported to a Pentium 200Mhz PC running Linux.

Typical Compilation:

  • gcc -O3 -I../SRC program.c -L.. -llcg

  • g77 -O3 -I../SRC program.F -L.. -llcg

SUN

SPRNG was been ported to a SUN workstation running SUN OS and SOLARIS.

Typical Compilation:

  • gcc -O3 -I../SRC program.c -L.. -llcg

  • f77 -fast -O3 -u -I../SRC program.F -L.. -llcg
